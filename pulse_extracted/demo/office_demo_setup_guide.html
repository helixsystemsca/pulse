Office Demo Setup Guide

Real hardware. Real inference. One ESP32, one beacon, your laptop. Zone-level tracking without trilateration — enough to fire a real ProximityPromptBanner during the pitch.

What You Need

✓ 1× ESP32 dev board
Flashed with pulse_gateway.ino. Acts as the only gateway — no nodes needed at camp. Does its own BLE scan.
✓ 1× BLE beacon
This is "Daniel's key fob". Any iBeacon/Eddystone device works. You'll register its MAC via seed script.
✓ Your laptop
Runs Mosquitto (MQTT broker) + position_engine_single.py. Also runs your FastAPI backend locally or points at Render.
✓ Your phone
Expo app running as Daniel. Shows ProximityPromptBanner when inference fires. This is the demo moment.
No RPI5 needed at camp. Your laptop IS the RPI5 for this demo. The position engine runs directly on your laptop. When you get home and add a second ESP32, trilateration kicks in and the dot moves — but the inference fires perfectly with just one gateway.

Evening 1 · Setup & Flash

1
Install Mosquitto MQTT broker on your laptop
This is the message broker the ESP32 publishes to. One command.
# Mac
brew install mosquitto
brew services start mosquitto

# Windows — download installer from mosquitto.org then:
net start mosquitto

# Verify it's running:
mosquitto_sub -t "test" -v &
mosquitto_pub -t "test" -m "hello"
# Should print: test hello
2
Find your laptop's local IP address
The ESP32 needs this to connect to Mosquitto. You'll paste it into the firmware.
# Mac / Linux
ipconfig getifaddr en0     # WiFi
ipconfig getifaddr en1     # sometimes WiFi is en1

# Windows
ipconfig | findstr "IPv4"

# Result will look like: 192.168.x.x — write this down
3
Flash the ESP32 gateway firmware
Open pulse_gateway.ino in Arduino IDE. Edit the 4 config lines at the top, then flash.
// Edit these 4 lines before flashing:

static const char* GATEWAY_ID    = "gw-demo-office-01";  // keep this exact
static const char* WIFI_SSID     = "YourLaptopHotspot"; // your laptop's hotspot name
static const char* WIFI_PASSWORD = "hotspot-password";  // hotspot password
static const char* MQTT_HOST     = "192.168.x.x";      // your laptop IP from step 2

// Leave everything else unchanged
After flashing, open Serial monitor at 115200 baud. You should see:
[GW] MAC address: AA:BB:CC:DD:EE:FF ← paste into Node GATEWAY_MAC
Write down this MAC address — you don't need it for camp but will for the node firmware at home.
4
Enable laptop hotspot and connect ESP32
Turn on your laptop's mobile hotspot (same SSID/password you put in firmware). The ESP32 Serial monitor should show WiFi connected and MQTT connected within 10 seconds.
# In a new terminal — watch for ESP32 MQTT messages:
mosquitto_sub -t "ble/rssi/#" -v

# You should see messages like:
# ble/rssi/gw-demo-office-01 {"gateway_id":"gw-demo-office-01","ts":1234,"readings":[...]}
If you see messages — the hardware chain is working. If not, check the MQTT_HOST IP in the firmware.
5
Find your beacon's MAC address
Hold the beacon near the ESP32. Watch the mosquitto_sub terminal. You'll see MACs appear in the readings array. Your beacon's MAC is the one that disappears when you move the beacon away.
# readings will look like:
{"mac": "AA:BB:CC:DD:EE:FF", "rssi": -55}  ← this one is close
{"mac": "11:22:33:44:55:66", "rssi": -85}  ← this one is far/noise

# Move beacon away — the strong one disappears. That's your beacon MAC.
# Write it down: AA:BB:CC:DD:EE:FF
6
Run the demo seed script
Creates the Pool zone, Hot Tub Boiler, overdue PM, Daniel user, and registers the gateway + beacon in your database.
# Set your beacon MAC first:
export DANIEL_BEACON_MAC="AA:BB:CC:DD:EE:FF"   # from step 5

# Run from backend folder:
cd backend
python -m scripts.seed_demo_office

# Output will include:
# ┌─────────────────────────────────────────────────────┐
# │  GATEWAY CREDENTIALS — copy these NOW               │
# │  Gateway UUID:   xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx│
# │  Ingest secret:  xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx│
# └─────────────────────────────────────────────────────┘
# Copy both values — you need them for step 7

Evening 2 · Wire & Test

7
Configure and run the position engine
Open position_engine_single.py. Paste the gateway UUID and secret from step 6. Run it.
# Edit these two lines at the top of position_engine_single.py:
GATEWAY_UUID   = "paste-your-gateway-uuid-here"
GATEWAY_SECRET = "paste-your-ingest-secret-here"

# Install deps if not already done:
pip3 install paho-mqtt requests numpy

# Run it:
python3 position_engine_single.py

# Should print:
# MQTT connected to localhost:1883 ✓
# Subscribed to ble/rssi/# — waiting for ESP32 data…
# FastAPI reachable at http://localhost:8000 ✓
8
Wire demo_routes.py into main.py (one Cursor session)
Add the demo router to your backend. Single prompt — under 2 minutes in Cursor.
# Cursor prompt:
# "In backend/app/main.py, add import and include for demo_routes.py.
#  from app.api.demo_routes import router as demo_router
#  app.include_router(demo_router, prefix='/api/v1')
#  Place near the other router includes. Don't change anything else."
9
Test the full inference flow
Hold beacon close to ESP32 for 60 seconds. Watch the position engine terminal.
# Position engine should print something like:
beacon AA:BB:CC:DD:EE:FF  rssi=-58dBm  dist=~1.2m  conf=0.42
→ FastAPI  positions=1  upserted=1  proximity_events_fired=1

# After ~60s near the boiler equipment:
→ FastAPI  positions=1  upserted=1  proximity_events_fired=1

# Check Pulse dashboard → /demo
# Inference card should appear with confidence rising toward 90%
# Expo app should show ProximityPromptBanner

The Demo Script (Pitch Day)

Setup before the meeting (5 min): Start laptop hotspot → plug in ESP32 → run position engine → open Pulse /demo on tablet → open Expo app on phone as Daniel → have beacon in your pocket.
①
"This is Daniel. He just came on shift."
Show the Expo app home screen. Daniel is logged in. His key fob is in your pocket — the beacon. Show the Pool zone on the dashboard map.
②
"He's doing his rounds. The hot tub boiler has a PM that's 3 days overdue."
Point to the PM overdue indicator on the map. Start walking toward the ESP32 (which is sitting near "the boiler"). Confidence meter is low.
③
"Pulse knows Daniel is near the boiler. It knows there's a PM due. It puts the pieces together."
Hold the beacon near the ESP32. Watch the confidence bar rise on the dashboard. Takes about 60 seconds to cross 90%.
④
"His phone just buzzed."
Hand them the Expo phone. ProximityPromptBanner is showing: "Are you servicing the Hot Tub Boiler? Monthly inspection · 3 days overdue." Let THEM tap Confirm.
⑤
"Work order updated. Zero manual entry."
Show the dashboard — work order auto-logged. "Your team currently spends X hours a week on paperwork like this. Pulse eliminates it."
Timing tip: The 60-second dwell requirement is configurable. For the demo, consider setting min_duration_seconds: 20 in the automation feature config so the inference fires faster. Change it back before showing it to a technical audience.
